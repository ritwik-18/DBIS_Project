SUMMARY
================================================================

What the algorithm shows:
  At equal ef_search, ACORN-1 and post-filter perform almost the
  same number of distance computations, yet ACORN-1 reaches
  dramatically higher recall. The paper claim is preserved:
  predicate-aware traversal makes each distance computation count.

Why ACORN-1 is slower in QPS despite less work:
  Buffer read counts reveal the substrate cost. ACORN-1 touches
  far more pages per query than post-filter (two-hop expansion).
  In PostgreSQL each ReadBuffer cycle costs ~1us, so the page
  access penalty dominates the distance computation savings.
  In FAISS, where pages are pointer-chases, this cost vanishes
  and the algorithmic win flows directly into QPS.
